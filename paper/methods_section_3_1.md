3.1 Pipeline Overview
Our quantization pipeline for OLMoE expert weights consists of three stages: routing-conditioned per-expert Hessian collection, RHT preprocessing followed by BlockLDLQ trellis quantization, and bitstream serialization. We adopt the trellis-coded quantization framework from QTIP [Tseng et al., 2024], adapting it to MoE expert weights via per-expert calibration. We do not use LUT fine-tuning; the analysis in Section 5.3 shows it provides negligible improvement when per-expert Hessians are used.

Hessian collection. For each expert e in each layer, we collect a per-expert input Hessian H_e = X_e^T X_e where X_e are the input activations of only the tokens that the router dispatches to expert e during the calibration pass. This is in contrast to existing MoE quantization work [MoEQuant, MoPEQ, EAQuant] which collects a single per-layer Hessian shared across all experts. We collect Hessians from a 2048-sequence × 1024-token sample of C4 [Raffel et al., 2020].

Random Hadamard Transform (RHT). Each weight matrix W and its corresponding Hessian H are transformed to an incoherent basis via random sign vectors and Hadamard matrices, following QuIP# [Tseng et al., 2024] and QTIP. The transformed weights W~ are approximately Gaussian-distributed (we measure aggregate kurtosis 3.85 across OLMoE expert layers, Section 5.4), making them well-matched to the trellis codebook. The transformed Hessian H~ encodes the same quadratic loss structure as H. RHT is invertible: at install time, we apply the inverse transform to recover W_q in the original basis.

BlockLDLQ trellis quantization. We compute the LDL decomposition H~ = LDL^T of the transformed Hessian, then run BlockLDLQ over column blocks of W~. At each block j, BlockLDLQ Viterbi-decodes the current residual target r[:, j] into trellis codes using QTIP's hybrid (HYB) bitshift codebook, appends those codes to the bitstream, and shifts all future residuals via r[:, k] ← r[:, k] − L[k,j] · e_j for k > j, where e_j = r[:, j] − W_q[:, j] is the quantization error on block j. The total loss decomposes as L(W_q) = Σⱼ D[j,j] · ||e_j||², which is exactly what each Viterbi step locally minimizes; the residual target r at column j is the conditionally optimal value for column j given the quantization committed for columns 0 through j-1.

Bitstream serialization. For each weight matrix, we save the trellis bitstream, the per-block start states, the RHT sign vectors, and the global scale factor. We do not save the dequantized weights themselves; W_q is reconstructed at install time by walking the bitstream through the codebook lookup and applying the inverse RHT. The on-disk representation occupies 3.3 GB for the full OLMoE-1B-7B model (vs 13.8 GB for fp16); see Section 4 for the breakdown.

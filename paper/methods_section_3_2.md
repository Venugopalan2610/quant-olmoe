### 3.2 Routing-Conditioned Per-Expert Hessian Collection

The defining feature of MoE inference is sparse routing: at each token, the router selects the top k experts (k=8 in OLMoE) from a pool of N (N=64) and dispatches the token's hidden state only to those experts. As a consequence, each expert sees a different subset of the calibration distribution. We exploit this by collecting a separate input Hessian for each expert from only the tokens that the router actually dispatches to it, rather than collecting a single Hessian per layer that aggregates over all tokens.

**Hook mechanism.** OLMoE's transformers implementation packs all 64 experts in each layer into a single `OlmoeExperts` module whose `forward` method iterates over the experts that received any tokens (the "expert hit set"). For each hit expert, it slices out the routed token subset, applies the fused `gate_up_proj[i]` followed by SiLU-and-multiply, then `down_proj[i]`, and scatters the results back via index-add. We install a per-instance monkey patch on `OlmoeExperts.forward` that mirrors this computation by construction while adding non-mutating side effects: for each routed token batch, we accumulate the input activations into per-expert Hessian buffers via in-place outer-product additions. The patched forward is mathematically identical to the original; the only differences are the auxiliary `addmm_` calls into the Hessian accumulators.

**Two Hessians per expert.** Each expert's MLP has two distinct input distributions:

- The **gate/up input** is the layer norm output of dimension H = 2048, which feeds the fused `gate_up_proj` matmul. We accumulate `H_e^{gate_up} = X_e^T X_e` where X_e is the (n_routed_tokens, H) matrix of routed inputs, in fp32 for numerical stability. Because `gate_proj` and `up_proj` are halves of a single fused weight that share their inputs exactly, a single Hessian serves both during quantization.
- The **down input** is the SiLU-gated intermediate of dimension I = 1024, which feeds `down_proj`. We separately accumulate `H_e^{down} = Y_e^T Y_e` where Y_e is the (n_routed_tokens, I) matrix of intermediate activations.

This gives 2N = 128 Hessians per layer (N for gate_up and N for down), totaling 16 × 128 = 2048 distinct expert Hessians for the full model. After the calibration sweep, each Hessian is normalized by its expert's token count and symmetrized (`H ← (H + H^T) / 2`) to correct for fp32 accumulation drift before being saved to disk.

**Calibration data.** We use 2048 sequences of 1024 tokens (totaling ~2.1M tokens) streamed from the C4 English training set [Raffel et al., 2020]. Sequences are tokenized with OLMoE's own tokenizer and packed contiguously across document boundaries with EOS separators. The Hessian buffers are kept on GPU in fp32; the calibration pass requires a single forward sweep over the full calibration set with the expert hook installed.

**Routing imbalance.** OLMoE's load-balancing loss does not produce perfectly uniform expert utilization. In our calibration sample, the ratio of the most-routed expert's token count to the least-routed expert's token count varies from 2.6× (layer 0) to 17.4× (layer 15), with a mean of 8.9× across the 16 layers. The imbalance grows in deeper layers, consistent with stronger expert specialization at deeper layers of the network. This is precisely the regime in which per-expert calibration should help: experts are seeing meaningfully different input distributions, so collapsing them into a single per-layer Hessian discards information.

**Controlled variable: attention Hessians.** In all configurations reported in this paper (per-expert, per-layer unweighted, per-layer weighted), we hold the attention Hessians fixed and identical to those collected with the per-expert pipeline. The ablation isolates the effect of routing-conditioned calibration on *expert weights only*. Attention quantization is held constant across configurations to remove it as a confounder, so any PPL gap observed between configurations is attributable solely to the change in expert Hessian collection.

**Per-layer baselines.** For our ablation in Section 5.1, we compare against two per-layer baselines that summarize the 64 per-expert Hessians as a single layer-level Hessian:

- **Unweighted mean:** `H^{layer}_{unweighted} = (1/N) Σ_e H_e`. Each expert contributes equally regardless of routing frequency. This baseline gives every expert's distinct distributional signature equal voice.
- **Token-weighted mean:** `H^{layer}_{weighted} = (Σ_e n_e · H_e) / (Σ_e n_e)`. High-traffic experts contribute proportionally more, which is mathematically equivalent to recollecting the Hessian on the full activation stream without partitioning by routing — the implicit calibration of MoEQuant, MoPEQ, and EAQuant.

We report both. The weighted baseline is the more standard comparison; the unweighted baseline tests whether per-expert calibration matters even when sparsely-routed experts are not penalized for having less data.
